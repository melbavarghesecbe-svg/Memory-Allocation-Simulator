What Is Memory Allocation?

Memory allocation is the process of assigning free memory blocks to incoming processes. A good allocation strategy improves utilization and reduces wasted space.

Goal: Fit processes quickly while keeping fragmentation low.

First Fit

Scan memory from left to right and choose the first block that can hold the process. It is simple and usually fast.

Example: Blocks: 100, 500, 200, 300 | Process: 212

First valid block is 500, so allocate there.

Best Fit

Choose the smallest block that is still large enough. This can reduce leftover space for each allocation.

Example: Blocks: 100, 500, 200, 300 | Process: 212

Valid blocks: 500, 300. Smallest is 300.

Worst Fit

Choose the largest available block that can fit the process. It keeps medium/smaller blocks free for future processes.

Example: Blocks: 100, 500, 200, 300 | Process: 212

Largest valid block is 500.

Fragmentation Notes

  • Internal fragmentation: Process gets a block bigger than needed, so some space inside that block is wasted.
  • External fragmentation: Total free memory exists, but it is broken into small separated holes.
  • Simple idea: Internal = waste inside a used block, External = waste between free blocks.

Real OS Connection

In real operating systems, memory managers must place process data in available RAM quickly and safely.

  • First Fit: Fast choice in many allocators because it stops early.
  • Best Fit: Tries to reduce leftover space, but may create many tiny holes.
  • Worst Fit: Keeps smaller blocks free, but can waste large memory chunks.
  • OS designers balance speed, memory usage, and fragmentation.
